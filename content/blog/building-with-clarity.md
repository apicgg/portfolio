---
title: Building With Clarity
date: 2026-03-05
summary: A short note on reducing complexity before adding more tools.
---

I like software that makes its tradeoffs obvious.

That usually means starting with the smallest system that can carry the work:

- plain files over hidden admin panels
- predictable structure over clever abstractions
- tools that are easy to remove later

When a project is still young, speed rarely comes from adding more layers. It comes from making the shape of the work legible.

If I cannot explain how content moves from source to production in a few sentences, the setup is already too heavy for most personal sites.
